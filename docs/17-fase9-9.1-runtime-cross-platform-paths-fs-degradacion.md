# Fase 9.1 — Runtime cross-platform (paths/FS + degradación elegante por SO)

## Diagnóstico de supuestos Windows-only

Hallazgos principales en runtime:

1. `cashdrawer.ipc.ts` usaba PowerShell RAW (`powershell.exe`) para modo impresora sin guardas por plataforma.
2. `serialport` se importaba en top-level, con riesgo de fallo de carga nativa y crash temprano en sistemas donde el módulo no esté disponible en runtime.
3. UI POS sugería `COM3` como valor por defecto incluso fuera de Windows.

## Cambios aplicados

### 1) Cashdrawer robusto por plataforma

Archivo: `app/electron/ipc/cashdrawer.ipc.ts`

- `serialport` pasa a carga lazy (`import('serialport')`) bajo demanda.
- Si `serialport` no está disponible en runtime, se devuelve error controlado (`SERIALPORT_UNAVAILABLE`) en vez de crash al arrancar.
- `mode: printer` ahora valida plataforma:
  - Windows: comportamiento existente (PowerShell RAW).
  - macOS/Linux: respuesta de error explícita y segura (`UNSUPPORTED_PLATFORM`) sin crash.

### 2) Default de puerto serial en UI según SO

Archivo: `app/renderer/src/pages/POS.tsx`

- Windows: mantiene `COM3` como default.
- macOS/Linux: default operativo genérico `/dev/ttyUSB0`.

## Compatibilidad por SO

### Windows x64

- Soporte completo esperado para flujo actual POS.
- Integraciones hardware (cajón por impresora RAW / serial) preservadas como flujo primario.

### macOS x64

- App no debe crashear por cashdrawer printer-mode Windows-only.
- Modo impresora de cajón queda degradado con mensaje explícito.
- Modo serial depende de disponibilidad real de `serialport` en runtime.

### Linux x64 (Debian/Ubuntu)

- Igual enfoque que macOS para degradación en printer-mode.
- Serial operativo si el módulo nativo está disponible y hay permisos de dispositivo.

## Limitaciones pendientes

- Integración de cajón por impresora RAW sigue siendo Windows-first (intencional en esta fase).
- No se aborda todavía packaging/firma/distribución por OS (fuera de alcance 9.1).
- Dependencias nativas (`better-sqlite3`, `serialport`) requieren build/runtime matching por plataforma en release final.

## Validación recomendada

- Smoke test de arranque en macOS/Linux con canales IPC de cashdrawer para confirmar degradación sin crash.
- Smoke de operaciones core (login, ventas, productos, caja, backups) en modo SQLite/MySQL según disponibilidad.
